White Clinic

  • 화이트크리닉이라는 청소 회사의 기사님 관리 프로그램(관리자 페이지) 개발을 목표로 한 팀 프로젝트입니다. 강사님이 외주로 수주한 프로젝트에 팀으로 참여하여 현재 진행 중입니다.

  • Skills
    Next.js, Nest.js(TypeORM), PostgreSQL, React, HTML, CSS, TypeScript

  • 담당한 역할
    - 고객 및 세척 품목 등록 기능: 고객 및 세척 품목을 통합 관리할 수 있는 등록 페이지를 구현했으며, 세척 품목의 금액은 자동 계산되며 필요시 직접 수정할 수 있도록 구성했습니다.
    - Auth 기능 구현 및 테스트 완료: RSA 키를 사용하여 access token과 refresh token 기반 인증을 구현하고, 스웨거로 로그인 기능 테스트를 완료했습니다.
    - Role 기반 접근 제어 설정: 관리자 페이지의 Role 기반 접근 제어를 통해 인증된 사용자만 접근할 수 있도록 보안을 강화하고, Role에 따른 권한을 설정하여 접근을 제어했습니다.
    - ERD 설계 및 테이블 생성: 초기 기획안을 바탕으로 PostgreSQL에 ERD를 설계하고 테이블을 생성했으며, 프론트엔드 팀과 협력하여 요구에 맞는 데이터 구조를 반영했습니다.

  • 현재 진행 중인 작업
    - DTO 및 서비스, 컨트롤러 구현: 기사님과 고객 관리 관련 페이지의 데이터 처리를 위해, 프론트엔드 타입을 기반으로 DTO, 컨트롤러, 서비스, 엔티티를 설계 및 구현 중입니다. 입력과 관리를 원활하게 하기 위한 데이터 구조를 다듬고 있습니다.
    - 기능 테스트 및 조정: Swagger를 사용해 새로 구현한 기능들을 테스트하며, 각 기능이 정상적으로 작동하는지 확인하고 필요한 조정을 진행하고 있습니다.

OTT / OTT-Admin

  • OTT 플랫폼과 관리자 페이지를 각각 구현하여 사용자와 관리자가 원하는 기능을 사용할 수 있도록 개발한 팀 프로젝트입니다. 주요 사용자 인터페이스는 React로, OTT 서비스의 백엔드는 Spring Boot로 구현하였으며, 관리자 페이지는 Next.js와 Express로 개발하였습니다.

  • Skills
    React, HTML, CSS, TypeScript, Next.js, Java, SpringBoot(JPA), JWT, MySQL, Node.js(Express)

  • 담당한 역할
    - Spring Boot를 활용한 백엔드 서버 개발과 ERD 설계를 통한 데이터베이스 구조 설계를 담당했습니다.
    - JPA를 사용하여 데이터를 관리하며, 비즈니스 로직을 구현했습니다.
    - JWT를 이용하여 사용자 인증과 보안을 구축했습니다.
    - Express.js와 Node.js를 사용하여 관리자 페이지의 서버를 개발하였습니다.

  • 개선점
    - Refresh Token 사용 미흡: 보안 강화를 위해 Access Token만 사용하는 대신 Refresh Token을 활용하여 더 안전한 인증 체계를 구축할 필요성을 느꼈습니다.
    - Role 기반 접근 제어 부족: 처음에는 관리자 페이지를 따로 만들 계획이 없었기 때문에 DB 구조나 보안에서 역할 기반 접근 제어가 미흡했습니다. 관리자 페이지를 포함한 경우, 이를 고려해 Role에 따라 접근을 제어할 수 있도록 설계하는 것이 더 나았을 것 같습니다.
